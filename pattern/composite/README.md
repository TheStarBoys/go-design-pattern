# 组合模式(Composite)

**定义**：

将对象组合成树形结构以表示“部分-整体”的层次结构。组合模式使得用户对单个对象和组合对象的使用具有一致性。

**本质**：

统一叶子对象和组合对象。

**优点**：

- 定义了包含基本对象和组合对象的类层次结构
- 统一了组合对象和叶子对象
- 简化了客户端调用
- 更容易扩展

**缺点**：

- 很难限制组合中的组件类型。

**建议在如下情况时选用外观模式**：

- 如果你想表示对象的部分一整体层次结构，可以选用组合模式，把整体和部分的操作统一起来，使得层次结构实现更简单，从外部来使用这个层次结构也容易。
- 如果你希望统一地使用组合结构中的所有对象，可以选用组合模式，这正是组合模式提供的主要功能。
